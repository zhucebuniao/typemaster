原理

怎么做到的？

Web Audio API + FFT = 频谱数据 → 字符画

1

加载音频

用 Web Audio API 的 AudioContext 加载音频文件，创建音频源。

2

FFT 分析

AnalyserNode.getByteFrequencyData() 获取频谱数据，0-255 的数组。

3

映射字符

频谱值映射到字符密度。低频→左边，高频→右边，振幅→字符高度。

4

实时渲染

requestAnimationFrame 循环，每帧重绘，产生流畅的动画效果。

// Web Audio API 获取频谱
const analyser = audioCtx.createAnalyser();
analyser.fftSize = 256;
const data = new Uint8Array(analyser.frequencyBinCount);

// 每帧获取频谱数据
function render() {
  analyser.getByteFrequencyData(data);
  // data[i] 就是 0-255 的频谱振幅
  drawASCII(data);
  requestAnimationFrame(render);
}
可视化模式

不同的视觉风格

同一种数据，不同的渲染方式

频谱柱状

经典频谱图，每个频率一个柱子，高度表示振幅。

波形

时域波形，展示音频的波形曲线。

环形

频谱围成一圈，中心向外围扩散。

矩阵

字符矩阵随音乐跳动，像黑客帝国。

文字

用歌词或文字随音乐变化。
